Wenn ich um einen Commit gebeten werde, darf ich die dafuer notwendigen Git-Befehle ausfuehren.
Bei UI-Labels moeglichst die Original-Bluesky-Bezeichnungen beibehalten.
Bei der Kernfunktionalitaet moeglichst dem Original folgen, sofern nichts anderes festgelegt ist.
Zum Committen erst fragen, wenn mindestens 5 Dateien geaendert wurden.
Bei Fragen immer zuerst antworten und erst danach Aktionen ausfuehren.
Wenn der Nutzer waehrend der Sitzung Aenderungen in den remind-me Dateien vornimmt, diese zuerst einlesen, bevor ich weiterarbeite.
Nach nennenswerten Erweiterungen/Aenderungen changelog-unreleased direkt aktualisieren.
Lesende Commandline-Befehle duerfen ohne Rueckfrage verwendet werden (z. B. `ls`, `grep` ohne Zustimmung).
Das gilt fuer alle lesenden Aktionen innerhalb des Repository-Roots (/home/michael/Entwicklung/BSky-Kampagnen-Bot).
Wenn der Nutzer mich zum Committen auffordert und keine Rueckfragen offen sind, darf ich eine passende Commit-Message selbst waehlen und committen.
`npm run build`, `npm run lint` und `npm run test` duerfen ohne Rueckfrage ausgefuehrt werden.
Nach groesseren Aenderungen `npm run build:all`, `npm run test:all` und `npm run lint:all` ausfuehren; groesser = neue Features, i18n-Umbau oder UI-Refactor.
